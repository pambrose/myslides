🔦 What is Redis?

  • REmote DIctionary Server
  • Fast NoSQL database written in ANSI C
  • An in-memory database -- all data is stored in RAM
  • Based on the key-value model
  • All access is via a key
  • Meant to be run as a network-atached server
  • Simple TCP protocol, which is implmented in many languages
  • Can act as a cache, database, pub-sub, and message broker

String Commands

SET/GET/DEL Commands

Set, get, and delete keys


> SET key1 "Hello"
OK
> SET key2 "World"
OK
> GET key1
"Hello"
> GET key2
"World"
> DEL key1 key2
(integer) 2
> GET key1
(nil)
>

Timed SET Commands

Expiring keys


> SET tempkey "Temp val with 10 secs of life" EX 10
OK
> SET cond1key "Set only if key does not exist" EX 10 NX
OK
> SET cond2key "Set only if key does exist" EX 10 XX
OK
>

MSET/MGET Commands

Set multiple values


> MSET key1 "Hello" key2 "World"
OK
> MGET key1 key2 nonexisting
1) "Hello"
2) "World"
3) (nil)
>

INCR/INCRBY Commands

Increment a value


> SET mykey "10"
OK
> INCR mykey
(integer) 11
> GET mykey
"11"
> INCRBY mykey 10
(integer) 21
> GET mykey
"21"
>

DECR/DECRBY Commands

Decrement a value


> SET mykey "10"
OK
> DECR mykey
(integer) 9
> GET mykey
"9"
> DECRBY mykey 5
(integer) 4
> GET mykey
"4"
>

List Commands

LPUSH/RPOP/LLEN Commands

Implementing a queue (first in, first out)


> LPUSH work:queue:ids 101
(integer) 1
> LPUSH work:queue:ids 237
(integer) 2
> LLEN work:queue:ids
(integer) 2
> RPOP work:queue:ids
"101"
> RPOP work:queue:ids
"237"
> LLEN work:queue:ids
(integer) 0
>

LPUSH/LPOP Commands

Implementing a stack (first in, last out)


> LPUSH work:queue:ids 101
(integer) 1
> LPUSH work:queue:ids 237
(integer) 2
> LPOP work:queue:ids
"237"
> LPOP work:queue:ids
"101"
>

LMOVE/LRANGE Commands

Moving items between lists


> LPUSH board:todo:ids 101
(integer) 1
> LPUSH board:todo:ids 273
(integer) 2
> LMOVE board:todo:ids board:in-progress:ids LEFT LEFT
"273"
> LRANGE board:todo:ids 0 -1
1) "101"
> LRANGE board:in-progress:ids 0 -1
1) "273"
>

BLPOP Command

Blocking list pop


> BLPOP work:ids 0
1) "work:ids"
2) "444"
> BLPOP work:ids 5
(nil)
(5.03s)
>

Set Commands

SADD/SMEMBERS Commands

Set creation and membership


> SADD user:123:favorites 347
(integer) 1
> SADD user:123:favorites 561
(integer) 1
> SADD user:123:favorites 742
(integer) 1
> SADD user:456:favorites 561
(integer) 1
> SMEMBERS user:123:favorites
1) "347"
2) "561"
3) "742"
>

SISMEMBER Command

Set membership: does user 123 like books 742 and 299?


> SISMEMBER user:123:favorites 742
(integer) 1
> SISMEMBER user:123:favorites 299
(integer) 0
>

SINTER Command

Set intersection: do users 123 and 456 have any favorite books in common?


> SINTER user:123:favorites user:456:favorites
1) "561"
>

SCARD Command

Set size: how many books has user 123 favorited?


> SCARD user:123:favorites
(integer) 3
>

Hashes

HSET/HGET/HGETALL Commands

Save a basic user profile as a hash


> HSET user:123 username martina firstName Martina lastName Elisa
(integer) 4
> HGET user:123 username
"martina"
> HGETALL user:123
1) "username"
2) "martina"
3) "firstName"
4) "Martina"
5) "lastName"
6) "Elisa"

Publish/Subscribe

SUBSCRIBE/UNSUBSCRIBE/PUBLISH Commands

Publish/Subscribe


> SUBSCRIBE topic1 topic2
subscribe from topic1: 1
subscribe from topic2: 2
message   from topic1: Hello
> PUBLISH topic1 "Hello"
(integer) 1
>

PSUBSCRIBE/PUNSUBSCRIBE Commands

Pattern-matching subscriptions


> PSUBSCRIBE weather.*
psubscribe from weather.*: 1
pmessage  from weather.*:weather.94528: 88
pmessage  from weather.*:weather.94513: 70
> PUBLISH weather.94528 88
(integer) 1
> PUBLISH weather.94513 70
(integer) 1
>
